# PopChat

Децентрализованный P2P мессенджер для локальной сети с end-to-end шифрованием, 
передачей файлов и голосовыми звонками. Не требует центрального 
сервера - каждый узел самодостаточен.

## Запуск

### Локально

```bash
# Python 3.12+
pip install uv
uv venv
uv sync
uv run py -m app
```

Приложение доступно на `http://127.0.0.1`, TCP P2P на порту `6767`, 
UDP discovery на `50000`.

## Использование

1. Запустите приложение на нескольких устройствах в одной сети
2. Укажите имя при первом запуске
3. Узлы автоматически обнаружат друг друга и установят соединение
4. Отправляйте сообщения, файлы и звоните

## Технологии

### Стек

| Компонент       | Технология                       |
| --------------- | -------------------------------- |
| Backend         | FastAPI + Uvicorn (async Python) |
| Шифрование      | PyNaCl (libsodium)               |
| База данных     | SQLite3                          |
| Сеть            | TCP + UDP + WebSocket            |
| Контейнеризация | Docker                           |

### Шифрование (E2E)

**Алгоритм:** NaCl Box — комбинация:
- **Curve25519** — эллиптическая кривая для обмена ключами (ECDH)
- **XSalsa20** — потоковый шифр для шифрования данных
- **Poly1305** — MAC для аутентификации (AEAD)

**Процесс:**
1. При первом контакте узлы обмениваются публичными ключами (`KEY_EXCHANGE`)
2. Из пары ключей (свой приватный + чужой публичный) формируется общий секрет
3. Каждое сообщение/файл/аудиофрейм шифруется перед отправкой
4. Ciphertext кодируется в base64 для передачи в JSON

**Верификация:** SHA256 от отсортированных публичных ключей → 8-значный 
safety number для защиты от MITM.

**Хранение ключей:**
- Приватный/публичный ключ: `keys/private.key`, `keys/public.key` (base64)
- Ключи пиров: SQLite таблица `keys`

## Транспорт

### UDP Discovery (порт 50000)
Периодический broadcast `HELLO`-пакетов на `255.255.255.255` каждые 3 секунды:
```json
{"type": "HELLO", "peer_id": "...", "name": "alice", "port": 6767}
```
При обнаружении нового соседа — установка TCP-соединения.

### TCP P2P (порт 6767)
- Постоянные соединения между соседями
- JSON-сообщения
- `TCP_NODELAY` для низкой задержки
- Idle timeout 20 сек → переподключение при необходимости

### WebSocket (порт 8001, путь /ws)
Уведомления фронтенда в реальном времени: новые сообщения, подтверждения 
доставки, входящие файлы, звонки.

## Маршрутизация

**Алгоритм:** Bellman-Ford (Distance-Vector).

Каждый узел хранит таблицу маршрутов `{destination → (gateway, ip, port, metric)}`.

- Узлы обмениваются маршрутами через `PEER_INFO`-пакеты
- Максимальная дистанция: **16 хопов**
- Каждый пакет содержит **TTL**, декрементируемый на каждом хопе
- **Reverse path filtering:** маршруты не анонсируются обратно через gateway

## Надёжность доставки

### Сообщения
1. Отправка → сохранение в БД (`delivered=false`)
2. Получатель расшифровывает → отправляет `ACK`
3. Отправитель получает ACK → `delivered=true`
4. Фоновый цикл каждые 10 сек пересылает недоставленные (до 3 попыток, TTL 1 час)

### Файлы
1. Файл разбивается на чанки по **32 КБ**
2. Каждый чанк шифруется и отправляется отдельно (`FILE_CHUNK`)
3. Получатель подтверждает каждый чанк (`FILE_ACK`)
4. При получении всех чанков — сборка + проверка **SHA256**
5. Недоставленные чанки автоматически пересылаются (до 3 попыток)

### Буферизация
Если маршрут до получателя недоступен, пакеты буферизуются (до 64 на адресат) и автоматически отправляются при появлении маршрута.

## Протокол

Все сообщения - JSON поверх TCP.

| Тип             | Назначение               |
| --------------- | ------------------------ |
| `MESSAGE`       | Текстовое сообщение      |
| `ACK`           | Подтверждение доставки   |
| `KEY_EXCHANGE`  | Обмен публичными ключами |
| `PEER_INFO`     | Анонс маршрутов          |
| `HELLO`         | UDP discovery            |
| `PING` / `PONG` | Измерение задержки       |
| `FILE_CHUNK`    | Фрагмент файла           |
| `FILE_ACK`      | Подтверждение фрагмента  |
| `CALL_OFFER`    | Инициация звонка         |
| `CALL_ANSWER`   | Ответ на звонок          |
| `CALL_END`      | Завершение звонка        |
| `CALL_AUDIO`    | Аудиофрейм               |
